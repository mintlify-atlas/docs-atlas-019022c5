## Get Historical Data

<api method="GET" url="/api/v1/statistics/historical-data" />

Retrieve complete historical data for a sensor type including current value, statistics, trend analysis, and chart data.

### Query Parameters

<ParamField query="greenhouseId" type="long" required>
  ID of the greenhouse
</ParamField>

<ParamField query="sensorType" type="string" required>
  Type of sensor (TEMPERATURE, HUMIDITY, CO2, LIGHT, etc.)
</ParamField>

<ParamField query="period" type="string" default="24h">
  Time period for analysis. Options: `24h` (hourly aggregation), `7d` (daily), `30d` (daily)
</ParamField>

### Response

<ResponseField name="greenhouseId" type="long">
  ID of the greenhouse
</ResponseField>

<ResponseField name="tenantId" type="long">
  ID of the tenant
</ResponseField>

<ResponseField name="sensorType" type="string">
  Type of sensor queried
</ResponseField>

<ResponseField name="unit" type="string">
  Unit of measurement
</ResponseField>

<ResponseField name="currentValue" type="number">
  Most recent sensor reading
</ResponseField>

<ResponseField name="currentValueTimestamp" type="string">
  Timestamp of current value (ISO 8601)
</ResponseField>

<ResponseField name="avgValue" type="number">
  Average value over the period
</ResponseField>

<ResponseField name="minValue" type="number">
  Minimum value recorded in the period
</ResponseField>

<ResponseField name="maxValue" type="number">
  Maximum value recorded in the period
</ResponseField>

<ResponseField name="medianValue" type="number" optional>
  Median value (if calculated)
</ResponseField>

<ResponseField name="trendPercent" type="number">
  Percentage change from average (positive or negative)
</ResponseField>

<ResponseField name="trendDirection" type="string">
  Trend direction: `INCREASING`, `DECREASING`, or `STABLE`
</ResponseField>

<ResponseField name="chartData" type="array">
  Array of time-series data points for charting
  <Expandable title="ChartDataPoint properties">
    <ResponseField name="timestamp" type="string">
      Timestamp of the data point
    </ResponseField>
    <ResponseField name="value" type="number">
      Aggregated value at this time
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="period" type="string">
  The period string used for this query
</ResponseField>

<ResponseField name="startTime" type="string">
  Start of the time range (ISO 8601)
</ResponseField>

<ResponseField name="endTime" type="string">
  End of the time range (ISO 8601)
</ResponseField>

### Example Request

<CodeGroup>

```bash curl
curl -X GET "https://api.example.com/api/v1/statistics/historical-data?greenhouseId=123456789&sensorType=TEMPERATURE&period=24h" \
  -H "Accept: application/json"
```

```javascript JavaScript
const params = new URLSearchParams({
  greenhouseId: '123456789',
  sensorType: 'TEMPERATURE',
  period: '24h'
});

const response = await fetch(
  `https://api.example.com/api/v1/statistics/historical-data?${params}`,
  {
    method: 'GET',
    headers: {
      'Accept': 'application/json'
    }
  }
);

const data = await response.json();
```

```python Python
import requests

response = requests.get(
    'https://api.example.com/api/v1/statistics/historical-data',
    params={
        'greenhouseId': 123456789,
        'sensorType': 'TEMPERATURE',
        'period': '24h'
    },
    headers={'Accept': 'application/json'}
)

data = response.json()
```

</CodeGroup>

## Get Sensor Trend

<api method="GET" url="/api/v1/sensors/stats/{sensorId}/trend" />

Calculate the trend (percentage change) for a sensor over a period.

### Path Parameters

<ParamField path="sensorId" type="string" required>
  Unique identifier for the sensor
</ParamField>

### Query Parameters

<ParamField query="tenantId" type="string" optional>
  ID of the tenant (for multi-tenant filtering)
</ParamField>

<ParamField query="period" type="string" default="24h">
  Time period. Examples: `24h`, `7d`, `30d`, `60m`
</ParamField>

### Response

Returns trend analysis for the sensor:

<ResponseField name="sensorId" type="string">
  ID of the sensor
</ResponseField>

<ResponseField name="currentValue" type="number">
  Most recent value
</ResponseField>

<ResponseField name="previousValue" type="number">
  Value at the start of the period
</ResponseField>

<ResponseField name="percentChange" type="number">
  Percentage change (positive or negative)
</ResponseField>

<ResponseField name="direction" type="string">
  Trend direction: `INCREASING`, `DECREASING`, or `STABLE`
</ResponseField>

<ResponseField name="period" type="string">
  Period used for calculation
</ResponseField>

### Example Request

<CodeGroup>

```bash curl
curl -X GET "https://api.example.com/api/v1/sensors/stats/TEMP_01/trend?period=24h" \
  -H "Accept: application/json"
```

```javascript JavaScript
const sensorId = 'TEMP_01';
const period = '24h';

const response = await fetch(
  `https://api.example.com/api/v1/sensors/stats/${sensorId}/trend?period=${period}`,
  {
    method: 'GET',
    headers: {
      'Accept': 'application/json'
    }
  }
);

const trend = await response.json();
```

```python Python
import requests

sensor_id = 'TEMP_01'
period = '24h'

response = requests.get(
    f'https://api.example.com/api/v1/sensors/stats/{sensor_id}/trend',
    params={'period': period},
    headers={'Accept': 'application/json'}
)

trend = response.json()
```

</CodeGroup>

### Example Response

```json
{
  "sensorId": "TEMP_01",
  "currentValue": 22.5,
  "previousValue": 21.8,
  "percentChange": 3.2,
  "direction": "INCREASING",
  "period": "24h"
}
```
